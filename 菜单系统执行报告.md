# 按键-菜单系统 执行报告

> 本文档总结本项目中**按键驱动(Key) + 菜单界面(Menu) + 屏幕显示(IPS200)** 的完整协作方案，
> 提炼出可移植的设计模式，供在其他嵌入式 MCU 工程中复用。

---

## 1. 系统概述

这是一套适用于**资源受限嵌入式平台**（TC264 TriCore）的非阻塞式菜单交互系统：

- **输入**：4 个独立按键（GPIO，上拉，低电平按下）
- **输出**：SPI 液晶屏（IPS200，8x16 字体）
- **控制流**：按键采样在中断中完成，菜单逻辑在主循环中轮询
- **状态模型**：基于**页面枚举 + 光标位置 + 编辑子状态**的有限状态机

核心原则：**中断只采集原始信号，所有决策放在主循环**。

## 3. 模块详解

### 3.1 按键驱动模块 (Key)

#### 设计要点

| 特性 | 实现方式 |
|------|----------|
| **去抖** | 5 次采样窗口（每 5ms 一次），≥4 次一致才确认状态 |
| **稳定状态输出** | `Key_Read(id)` 返回 0/1，由中断更新，主循环读取 |
| **长按检测** | 在稳定态基础上累加节拍，`KEY_LONG_PRESS_TICKS=24`（约 600ms） |
| **长按事件** | `Key_ReadLongPress(id)` 单次触发，读取后自动清零 |

#### 可移植代码骨架

```c
// Key.h ─── 对外接口
void  Key_Init(void);
void  Key_Sample(void);        // 中断中 5ms 调用
uint8 Key_Read(uint8 id);      // 0=松开, 1=按下
uint8 Key_ReadLongPress(uint8 id);  // 长按事件, 读后清零

// Key.c ─── 核心实现模式
#define SAMPLE_WINDOW  5       // 5 次采样窗
#define LONG_PRESS_TICKS 24    // 约 600ms

static uint8 s_cnt[4];         // 窗口内按下计数
static uint8 s_down[4];        // 稳定态输出
static uint8 s_hold[4];        // 长按累积节拍
static uint8 s_long_evt[4];    // 长按事件锁存

void Key_Sample(void)
{
    // 1. 采集原始电平 (假设低电平=按下)
    for (int i = 0; i < 4; i++) {
        if (gpio_get_level(key_pin[i]) == 0) s_cnt[i]++;
    }

    // 2. 窗口满则更新稳定态
    static uint8 tick = 0;
    if (++tick >= SAMPLE_WINDOW) {
        tick = 0;
        for (int i = 0; i < 4; i++) {
            s_down[i] = (s_cnt[i] >= 4) ? 1 : 0;
            // 长按逻辑
            if (s_down[i]) {
                if (++s_hold[i] >= LONG_PRESS_TICKS && !s_latched[i]) {
                    s_long_evt[i] = 1;
                    s_latched[i] = 1;
                }
            } else {
                s_hold[i] = 0;
                s_latched[i] = 0;
            }
            s_cnt[i] = 0;
        }
    }
}
```

### 3.2 菜单状态机 (Menu)

#### 三要素

每个菜单页面由以下三者共同定义：

```
s_ui_page  (ui_page_t 枚举)  →  决定：画什么 + 按键怎么响应
s_xx_item  (uint8 光标行号)  →  决定：">" 画在哪一行
s_xx_state (编辑子状态枚举)   →  决定：当前是选择态还是编辑态
```

#### 按键语义表（可复用约定）

| 按键 | 通用含义 | 选择态行为 | 编辑态行为 |
|------|----------|-----------|-----------|
| **KEY1** | **下 / 减** | 光标下移一行 | 参数值减小 |
| **KEY2** | **确认** | 进入子菜单 / 进入编辑态 | 确认修改，返回选择态 |
| **KEY3** | **上 / 加** | 光标上移一行 | 参数值增大 |
| **KEY4** | **返回** | 返回上一级菜单 | 放弃编辑，返回上级 |

#### 页面状态机模板（最通用的模式）

```c
// ─── 状态枚举 ───
typedef enum {
    STATE_SEL,    // '>' 选择行
    STATE_EDIT,   // '-' 编辑值
} edit_state_t;

// ─── 字段枚举 ───
typedef enum {
    FIELD_A = 1,
    FIELD_B,
    FIELD_C,
    FIELD_COUNT = FIELD_C,
} field_t;

// ─── 状态变量 ───
static volatile edit_state_t s_state = STATE_SEL;
static volatile uint8 s_item = 1;

// ─── 按键处理 ───
if (s_state == STATE_SEL) {
    if (KEY1) { s_item++; if (s_item > FIELD_COUNT) s_item = 1; }
    if (KEY3) { s_item--; if (s_item < 1) s_item = FIELD_COUNT; }
    if (KEY2) { s_state = STATE_EDIT; }
}
else {  // STATE_EDIT
    if (KEY1 || long_press_KEY1) { adjust(s_item, -1); }
    if (KEY3 || long_press_KEY3) { adjust(s_item, +1); }
    if (KEY2) { s_state = STATE_SEL; }
}
if (KEY4) { /* 返回上级页面 */ }
```

### 3.3 屏幕绘制规则

每个 `draw()` 函数的固定布局：

```c
static void my_page_draw(void)
{
    // 1. 设置字体
    ips200_set_font(IPS200_8X16_FONT);

    // 2. 绘制标题（固定行 0）
    ips200_show_string(0, 0 * LINE_H, "PAGE_TITLE");

    // 3. 绘制各字段（固定行 1~N）
    ips200_show_string(16, 1 * LINE_H, "LABEL1:");
    ips200_show_float(x, 1 * LINE_H, value1, 6, 2);
    ips200_show_string(16, 2 * LINE_H, "LABEL2:");
    ips200_show_int(x, 2 * LINE_H, value2, 5);

    // 4. 绘制光标（" >" 或 " -"）
    char cursor = (s_state == STATE_EDIT) ? '-' : '>';
    ips200_show_char(0, (1 + s_item) * LINE_H, cursor);

    // 5. 可选：页脚签名
    draw_signature_bottom();
}
```

布局公式：
- 行号从 0 开始
- 像素 y = 行号 × **行高**（本项目使用 15px）
- 标签列 x = 16（缩进）；值列 x = 16×N（视标签长度定）
- 光标列 x = 0

## 4. 关键设计决策与理由

| 决策 | 理由 |
|------|------|
| **中断采集 + 主循环消费** | 避免按键逻辑阻塞中断；去抖窗口不影响主循环实时性 |
| **5ms 采样 × 5 次 = 25ms 去抖** | 机械按键抖动通常 < 20ms，25ms 窗口足够可靠且响应可接受 |
| **边沿检测在 Menu 层自己做** | Key 模块只输出"稳定电平"，Menu 层决定"上升沿触发"，解耦清晰 |
| **页面用枚举而非函数指针** | MCU 资源有限，switch-case 可预测、易调试，跳转开销低 |
| **长按加速用循环执行 N 次** | 无需改造 Key 模块，在 Menu 层用简单 loop 实现"按住快速调值" |
| **全局变量直读直写** | 嵌入式场景下避免复杂消息队列，参数修改即时生效，简单可靠 |
| **每次按键后 ips200_clear()** | 避免局部擦除的逻辑复杂度，刷屏速度足够（~20fps） |

## 5. 组合键实现模式

```c
// Menu_Select() 开头，优先于单键检测
if (k2_now && k4_now) {         // KEY2+KEY4 同时按住
    if (!k24_latched) {
        k24_latched = 1;
        menu_show_flag = !menu_show_flag;
        ips200_clear();
    }
    return;  // 跳过单键处理
} else {
    k24_latched = 0;
}
```

要点：
- **双按键同时按下检测**：读取两个按键的当前稳定态
- **锁存变量**（`k24_latched`）：防止长按重复触发，只有都松开后才能再次触发
- **优先级最高**：放在 `Menu_Select()` 最前面，组合键生效时跳过所有单键逻辑

## 6. 在新工程中实施的步骤

### 第一步：移植按键驱动

1. 拷贝 `Key.h` / `Key.c`，修改 GPIO 引脚宏定义
2. 配置一个 5ms 定时器中断，在 ISR 中调用 `Key_Sample()`
3. 初始化时调用 `Key_Init()`

### 第二步：搭建菜单框架

1. 定义页面枚举 `ui_page_t`
2. 定义页面状态变量（当前页、光标行号、编辑状态）
3. 实现 `User_Interface()` — 根据 `s_ui_page` 分发到各 draw 函数
4. 实现 `Menu_Select()` — 按页面分类处理按键事件

### 第三步：实现具体页面

按以下清单逐项完成：

```
[ ] 添加枚举值到 ui_page_t
[ ] 添加状态变量（static volatile）
[ ] 创建 draw 函数（绘制标签、数值、光标）
[ ] 在 User_Interface() 中加入 else-if 分支
[ ] 在 Menu_Select() 中加入 else-if 分支处理按键
[ ] 创建数值调节辅助函数（带限幅）
[ ] 确认 KEY4 返回路径正确
[ ] 测试页面前后切换、数值编辑、边界值
```

### 第四步：集成到主循环

```c
while (1) {
    Menu_Select();     // 处理按键
    User_Interface();  // 刷新屏幕
    // 其他周期性任务...
}
```

## 7. 典型 Bug 与防范

| 现象 | 根因 | 对策 |
|------|------|------|
| 按键反应慢半拍 | 边沿检测用了稳定态而非上升沿 | 检查 `k1_now && !k1_last` 逻辑 |
| 菜单页面混乱 | `else-if` 链中漏了 `else`，多个页面同时响应 | 确保每个页面分支前都有 `else` |
| 返回后光标位置不对 | 进入子页时未重置 item/state | 进入子页时显式初始化 `s_xx_item = 1; s_xx_state = SEL;` |
| 长按不生效 | 长按事件被边沿检测的门控挡住了 | 编辑态下长按和短按分开处理，互不阻塞 |
| 修改参数后无效 | 改了配置表但未调用 apply 函数 | 在 KEY2 确认时调用 `ttop_funtion_mode_once_apply()` 或等效的初始化函数 |
| 组合键误触发 | 未做锁存，长按导致反复触发 | 加 `_latched` 锁存变量，松开才能再次触发 |

## 8. 本项目的性能参考

| 指标 | 数值 |
|------|------|
| 按键采样周期 | 5ms（定时器中断） |
| 按键去抖输出周期 | 25ms |
| 长按判定阈值 | ~600ms（24 个去抖周期） |
| 菜单轮询频率 | ≈ 主循环频率（与 `gnss_data_parse()` 等共享） |
| 屏幕刷新时机 | 每次按键后 `ips200_clear()` + 下次主循环 `User_Interface()` |
| IPS200 通信 | SPI，8x16 字体，行高 15px |
| 状态变量存储 | RAM（static volatile），掉电不保存 |
| 参数持久化 | 组合键 KEY1+KEY3 保存到 Flash |
